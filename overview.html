Wine Graph

Core wine data shape

Clean and simple view of how area, producer, wine, and their references relate.

Area (AVA, AOC, DOC, etc.)

A simple hierarchy for place and appellation context.

  • name
  • country / region / subregion
  • producers / wines

Producer

The source-of-truth owner for canonical wine identity.

  • name
  • story / profile
  • tech sheets
  • website / email / phone
  • social media
  • wines

Wine

The canonical wine record used across the graph.

  • name
  • vintage
  • varietal / grape
  • harvest / bottling / release dates
  • aging
  • color / type / closure / bottle data

Wine Ref

Links the canonical wine to outside systems or ingested source data.

  • source name
  • source id
  • sku or handle
  • match status
  • import metadata

How the core data connects

These relationships are intentionally simple, so the model stays easy to explain, extend, and connect.

Area → Producer

Geographic home and production context.

Producer → Wine

A producer publishes their catalog of wines.

Wine → Wine Ref

A canonical wine can connect to many external records. The core concept around wine identity and linking across retailers and producers ensuring consistency and traceability.

Wine Graph

User types and functionality

This is more product-facing than technical. It shows who the main user types are and what each one does in the system.

Visitor / Consumer

Lightweight discovery user.

  • search wines and producers
  • browse regions and areas
  • view retailer availability
  • save or follow later

Enthusiast

More discerning and involved user.

  • save favorite wines, producers, retailers
  • write personal tasting notes
  • compare notes with producers and somms
  • track wine trends
  • build their own wine profile

Sommelier

Wine superuser, taster, expert.

  • search wines and producers
  • store wines for tracking and analysis
  • provide somm level tasting notes for canonical wines

Producer

Canonical data owner and publisher.

  • create and manage wines
  • maintain a profile to ensure accuracy and completeness
  • publish wine catalog
  • track downstream presence
  • connect with retailers to improve discoverability and penetrate new markets

Retailer

Inventory and discovery operator.

  • connect POS or commerce source
  • sync inventory
  • review matches
  • improve discoverability

Simple platform framing

  • Producers improve the graph by publishing accurate canonical wine data.
  • Retailers improve the graph by linking real inventory to that canonical data.
  • Consumers benefit from cleaner discovery and better availability signals.
  • Sommeliers benefit from more accurate wine quality data and providing expert level tasting notes and reviews.
  • Admin / Operators keep quality high when data is messy or ambiguous.
